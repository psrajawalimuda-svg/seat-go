

# Application Review & Refinement Plan

## Issues Found

### Critical Build Errors
1. **`src/integrations/supabase/types.ts` is corrupted** — Contains binary/UTF-16 encoded data instead of valid TypeScript. This file is auto-generated and cannot be edited manually. It needs to be regenerated by running a database schema sync.
2. **Missing `Star` import in `UserDashboard.tsx`** — Line 244 uses `<Star>` but it's never imported from `lucide-react`.

### Logic Bugs
3. **DriverHome.tsx: `t.date` doesn't exist** — The `toTrip()` helper returns `departureDate`, not `date`. Line 58 filters `t.date === todayDate` which always evaluates to `undefined === "2026-03-26"`, so today's missions never match.
4. **`toBooking()` hardcodes `destination: "Unknown"`** — The booking doesn't store destination, but it could be inferred from the trip's `route_name`.
5. **`Booking` interface mismatch** — `BookingContext` sets extra fields (`id`, `status`, `bookedAt`) on the `Booking` type from `shuttle-data.ts` which doesn't define them. This causes silent type errors.

### UX Issues
6. **UserDashboard phone-based identity** — Uses `localStorage` for user identity (insecure, easily spoofable). Noted for future auth implementation.
7. **SearchResults doesn't filter by date** — All trips are shown regardless of selected travel date.
8. **TrackTicket search logic is weak** — Any input containing "SG-" matches any booking, not the actual reference.

## Refinement Plan

### Step 1: Fix types.ts (Database Sync)
Trigger a database migration (no-op or minor change) to force regeneration of the `types.ts` file. This resolves the build error.

### Step 2: Fix Missing Import in UserDashboard.tsx
Add `Star` to the lucide-react import list.

### Step 3: Fix DriverHome.tsx Date Filtering
Change `t.date` to `t.departureDate` on line 58 so today's missions filter works correctly.

### Step 4: Extend Booking Interface
Add optional fields (`id`, `status`, `bookedAt`, `destination`) to the `Booking` interface in `shuttle-data.ts` so BookingContext and UserDashboard can set them without type errors.

### Step 5: Fix toBooking Destination
Use `trip.route_name` (passed as parameter) instead of hardcoded `"Unknown"` for the destination field in `toBooking()`.

### Step 6: Filter SearchResults by Date
Add date filtering in `SearchResults.tsx` so only trips matching the selected travel date are shown. Fall back to showing all if no `departure_date` is set on the trip.

### Step 7: Improve TrackTicket Search
Make the ticket search match by booking ID or generate the same `SG-XXXX` reference code and match against it, instead of the current loose substring check.
